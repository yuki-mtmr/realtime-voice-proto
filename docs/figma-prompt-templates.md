# Figma実装プロンプトテンプレート

## Step 1: デザイントークン取得（最初に1回）

```
Figma MCPで以下のファイルのバリアブルとスタイルを取得してください。

Figma URL: [ファイルのURL]

取得後、以下を実行してください。
1. カラー、タイポグラフィ、スペーシングのバリアブルを一覧化
2. tailwind.config.ts にマッピングを追加
3. 実装管理ドキュメントのデザイントークンセクションを更新
```

## Step 2: 画面の構造把握（画面着手時に1回）

```
Figma MCPで以下の画面の構造を取得してください。

Figma URL（dev）: [画面全体のフレームを選択した状態のURL]

以下を整理してください。
1. この画面のセクション構成（中間フレームの一覧）
2. 各セクションのノードID
3. 使われている共通コンポーネント
4. 実装管理ドキュメントの該当画面のセクション表を更新

この段階では実装しないでください。構造の把握のみ行います。
```

## Step 3: セクション実装（セクションごとに繰り返す）

```
Figma MCPで以下のセクションを取得し、実装してください。

Figma URL（dev）: [セクションの中間フレームを選択した状態のURL]

実装ルール:
1. tailwind.config.ts に定義済みのデザイントークンを使用すること
2. 既存の共通コンポーネントがあれば再利用すること
3. 画像やアイコンがあればdownload_assetsで取得すること
4. ハードコードされた色・余白値は使わないこと
5. 実装完了後、実装管理ドキュメントのステータスを更新すること
```

## 共通コンポーネント作成

```
Figma MCPで以下のコンポーネントの詳細を取得し、共通コンポーネントとして実装してください。

Figma URL（dev）: [コンポーネントのURL]

要件:
1. すべてのバリアントを確認し、Propsとして定義すること
2. デザイントークンを使用すること
3. 実装管理ドキュメントの共通コンポーネント一覧を更新すること
```
